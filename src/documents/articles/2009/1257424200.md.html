에잇 버그 투성이…

급조한 듯한 문법의 Action Script트 뿐만 아니라 Flex Builder의 버그와 빌드 속도등 개발자의 기운을 빠지게 하는 요소가 많습니다. 한숨이 나옵니다.

오늘은 ArrayCollection관련 코드를 짜는데 어이 없는 일이 발생했습니다.  디버깅을 해도 맞는 코드가 분명한데 특정 라인을 실행한후에 filterFunction이 풀려버리는 이상한 현상을 발견했습니다. ArrayCollection의 설계상 한계(뭔게 제가 모르는)라고 해도 너무 어이가 없어서 웃음도 나오질 않아요.

예를 들면 다음과 같은 코드

var a:ArrayCollection = new ArrayCollection(.....);
a.필터적용
a.리프레시
[Bindable]
var b:ArrayCollection

b = a
b.length//필터가 풀린 개수가 반환됨.

그래서 고민끝에 2시간 삽질 끝에 해결한 코드는 다음과 같습니다.

var a:ArrayCollection = new ArrayCollection(.....);
[Bindable]
var b:ArrayCollection

b = a
b.필터적용

b.리프레시
b.length//필터가 적용된 개수가 반환됨.

다시금 강조하지만 버그가 아닌 ArrayCollection의 설계상 한계라고 해도 짜증나는 일입니다.

 당장이라도 집어 치우고 싶지만 flex 단가가 높아서 돈을 좀더 준다는 말에 DB, ibatis, spring, bladeds, flex에 이르는 코딩을 하고 있습니다. 서버개발자나 클라이언트 개발자의 협업이 없으니 재미가 떨어지고 빨리 지치는 것 같습니다.
