단순히 다음과 같이 하면 됩니다.

var e:Error = new Error();
caller = e.getStackTrace().split("\n", 3)[2];

그런데 이 것으로는 완벽하지 않습니다. release build에서는 callStack정보를 얻어올 수 없는 것 같습니다. 게다가 handling하지 않은 Error는 씹어 버리기 때문에 e.getStackTrace()의 값을 조작하는 것은 대박 삽질로 연결됩니다. 실제로 대박 삽질 했습니다.-_-;;;

e.getStackTrace()를 사용하는 이용 코드는 반드시 handle하는 습관이 있어야 겠습니다. 다음처럼 말입니다. 이제는 너무 옜날 일이라 다 잊어 먹었는데 Visual Studio로 개발하다가 몇달 후 release Build를 시도할 때마다 점점 미쳐갔었던 어릴적 풋풋한 기억이 생각나네요.

try{
  var e:Error = new Error();
  caller = e.getStackTrace().split("\n", 3)[2];
}catch(e:Error){
  caller = "씹혔어요";
}
