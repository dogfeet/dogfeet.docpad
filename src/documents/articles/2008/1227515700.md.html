사실 개발자용 블로그에는 소스코드를 예쁘게 보여주는 플러그인이 필수인데, 티스토리에서는 아직 해당 플러그인이 공식적으로 지원되지 않더군요. pupustory님의 dp.SyntaxHighlighter를 이용한 코드 하이라이트글을 참고하여 수동으로 플러그인을 (스킨에 구겨넣어서)설치해놨습니다.

코드를 예쁘게 보여주는 엔진은 SyntaxHighlighter라는 프로그램인데 구글 코드에서 활동중인 공개프로그램인가 봅니다. 다운로드를 받아 플래시 파일(swf), 코드파일(js), 스타일시트(css)를 스킨 이미지 업로드를 통해 업로드 하고 다음과 같이 스킨파일을 약간 수정해줍니다.

Head태그 내에 스타일 시트 링크하기

<link type=&quot;text/css&quot; rel=&quot;stylesheet&quot; href=&quot;./images/SyntaxHighlighter.css&quot; />

Body의 닫는 태그 바로 앞부분에 Script코드 추가하기

<pre class=&quot;html&quot; name=&quot;code&quot;>
<!-- google code for syntaxhighlight -->
<script class=&quot;javascript&quot; src=&quot;./images/shCore.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushCpp.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushCSharp.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushCss.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushDelphi.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushJava.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushJScript.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushPhp.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushPython.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushRuby.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushSql.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushVb.js&quot;></script>
<script class=&quot;javascript&quot; src=&quot;./images/shBrushXml.js&quot;></script>
<script class=&quot;javascript&quot;> dp.SyntaxHighlighter.ClipboardSwf = './images/clipboard.swf'; dp.SyntaxHighlighter.HighlightAll('code'); </script>
<!-- google code for syntaxhighlight --> 
</body>
</body>

그리고 코드는 다음과 같이 작성합니다.

<pre class=&quot;cpp&quot; name=&quot;code&quot;>
cout << &quot;Test Ok&quot; << endl;
</pre>

현재 위의 코드는 Html Tag를 &lt;나 &gt;로 변환하고(cout &lt;&lt; &quot;Test Ok&quot; &lt;&lt; endl;) 저장하였습니다. 잘 나오는것 같네요.

사실 위 처럼 직접 태그를 쓰려니 기존에 사용하던 위지윅 편집기가 pre사이에 br태그를 집어넣어서 편집할 때 마다 손을 봐줘야 하는 상황이 발생합니다. 게다가 pre태그 사이에 소스코드중에 html 태그들과 충돌이 발생하는 기호(예를 들어 <, >)들은 &lt; &gt;와 같이 변환해 주는 과정이 필요한데(위지윅에서는 자동으로 될테지만 역시 줄바꿈 충돌 문제…), 이는 손수 해줘야 할 것 같습니다.

따라서 입력하기 전에는 HTML Encoder를 사용하여 html 기호들에 대한 encoding를 해야 합니다. 그리고 위지윅 에디터와 관려한 문제는 우선 수정하기가 힘들어서 아예 편집 모드를 HTML기본으로 변경하였습니다. 그러고 나니 위키 형식의 포매터가 있었으면 하는 바램이 다시한번 드네요 ^^;;
